# 安全策略

TakeFive 是本地优先桌面应用。涉及本地数据库、Tauri IPC、文件路径、更新与签名、权限提升、命令执行或敏感信息泄露的问题，都应按安全问题处理。

## 支持范围

| 版本 | 状态 |
| --- | --- |
| `main` / `0.1.x` | 预发布阶段，接收安全修复 |
| 更早的开发快照 | 不单独维护 |

当前尚未发布面向普通用户的签名安装包，因此不存在官方二进制下载渠道。第三方构建不属于本项目的安全支持范围。

## 报告漏洞

优先使用 GitHub 仓库 **Security** 页面中的 **Report a vulnerability** 私密报告入口（仓库启用 Private vulnerability reporting 后可用）。请提供：

- 受影响版本、提交或构建来源；
- 操作系统和必要环境信息；
- 可复现步骤与实际影响；
- 最小化的日志、截图或验证代码；
- 你建议的修复方向（如有）。

请先移除用户名、本地路径、提醒内容、数据库数据、访问令牌和签名材料。不要在公开 Issue、讨论区或 PR 中发布可直接利用的漏洞细节。

如果私密报告入口尚未启用，请先创建一个不包含利用细节的公开 Issue，请求维护者提供私密沟通渠道。收到报告后，维护者会先确认问题、评估影响，并在修复可用后协调披露。
