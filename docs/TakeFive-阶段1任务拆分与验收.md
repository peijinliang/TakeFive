# 摸个鱼 · TakeFive 阶段 1 任务拆分与验收

**版本：** V0.1  
**日期：** 2026-07-14  
**阶段目标：** 建立可恢复、可防重、可解释的本地提醒调度内核

---

## 1. 目标闭环

```mermaid
flowchart LR
    Rule[领域规则计算] --> Candidate[下一候选事件]
    Candidate --> Claim[SQLite 原子认领]
    Claim --> Policy[暂停/勿扰/系统状态策略]
    Policy --> Delivery[投递端口]
    Delivery --> Transition[Occurrence 状态转换]
    Transition --> History[历史与原因持久化]
    History --> Reconcile[启动/唤醒/时间变化对账]
    Reconcile --> Rule
```

阶段 1 完成的判断标准不是“能定时弹一次”，而是同一计划事件在并发、重启、时间回拨和重复对账下都只能被认领一次，并能说明未展示原因。

## 2. 并行工作流

| 工作流 | 负责内容 | 主要产物 | 依赖 |
| --- | --- | --- | --- |
| A. 领域核心 | Reminder、Occurrence、状态机、固定/间隔/一次性规则 | `crates/domain` | 无 |
| B. 本地持久化 | SQLite migration、Repository、事件唯一约束、事务认领 | `crates/persistence-sqlite` | 无，集成时映射领域类型 |
| C. 平台事件 | 休眠、锁屏、时间、时区和显示器变化事件 | `platform.rs` | Windows API |
| D. 调度器 | 队列、到期认领、策略、对账与可注入端口 | `crates/scheduler` | A、B 的稳定接口 |
| E. 桌面应用服务 | IPC、通知投递、托盘状态、最小提醒管理 | `apps/desktop/src-tauri` | C、D |
| F. 验证与文档 | 跨模块测试、Release 运行、需求覆盖矩阵 | `docs`、测试报告 | A-E |

前三项可以并行开发，D 在接口落定后集成，E 不直接操作 SQL 或自行计算时间。

## 3. 模块契约

### 3.1 领域层

- 输入：规则、参考时刻、时区和生效约束。
- 输出：稳定候选事件与 `occurrence_key`。
- 负责状态转换是否合法。
- 不依赖 SQLite、Tauri、Win32 和真实系统时间。

### 3.2 持久化层

- 输入：待保存实体、候选事件认领请求和事务用例。
- 输出：已插入、已存在、冲突或存储错误。
- 唯一约束是防重复最后防线。
- 不决定暂停、勿扰和补发策略。

### 3.3 平台层

- 输入：原生系统消息。
- 输出：统一平台事件和当前能力状态。
- 不直接改 Reminder/Occurrence。
- 不把启发式全屏判断包装成确定事实。

### 3.4 调度器

- 只通过端口访问时钟、存储、系统上下文和投递通道。
- 内存队列可随时由数据库与规则重建。
- 所有投递前先认领，所有决策保存 reason code。
- 对相同 `reminder_id + occurrence_key` 幂等。

## 4. 阶段 1 验收场景

| 编号 | 场景 | 证据 |
| --- | --- | --- |
| S1-01 | 两个并发调度器认领同一事件 | SQLite 集成测试仅一个成功 |
| S1-02 | 系统时间回拨后重复对账 | occurrence 唯一键阻止二次认领 |
| S1-03 | 固定间隔执行多轮 | 候选时间由 anchor/index 计算，无累计漂移 |
| S1-04 | 延后后计算下一周期 | 原始规则锚点保持不变 |
| S1-05 | 事件完成后再次延后 | 状态机拒绝非法转换 |
| S1-06 | 应用重启 | 从数据库恢复未终态事件和下一候选 |
| S1-07 | 休眠期间错过健康循环 | 对账生成 Missed/Ignored 原因，不批量投递 |
| S1-08 | 一次性提醒在 24 小时内错过 | 生成一次补发候选并注明原计划时间 |
| S1-09 | 勿扰或暂停 | 不投递但保存可解释 reason code |
| S1-10 | 数据库写入失败 | 不投递无法认领的事件，进入保护状态 |

## 5. 需求覆盖边界

阶段 1 覆盖：

- 固定时间、间隔循环、一次性提醒的可靠内核。
- Occurrence 与投递尝试数据基础。
- 完成、跳过、延后、未处理等状态基础。
- 休眠、锁屏、系统时间和时区变化事件输入。
- 重启恢复、并发防重和错误保护。

阶段 1 不宣称完成：

- 完整四页面和首次引导。
- 全屏休息窗口与声音系统。
- 完整勿扰编辑器、多提醒 60 秒动态聚合。
- 历史图表、主题、备份导入导出。
- macOS 真机适配、签名和公证。

这些仍属于完整需求目标，只是按照依赖顺序进入后续阶段，不从范围中删除。
