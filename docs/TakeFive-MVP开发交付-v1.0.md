# 摸个鱼 · TakeFive MVP 开发交付

**版本：** V1.1  
**日期：** 2026-07-15  
**状态：** MVP 主功能开发完成，待 Windows 桌面真机验收

---

## 1. MVP 范围

本版本只包含三个产品域：

1. 设置
2. 提醒
3. 桌面悬浮框

本版本不开发今日时间线、历史记录、统计报表、主题、账号、同步、导入导出或备份界面。底层已有但未进入这三条用户链路的能力不继续扩展。

---

## 2. 已完成功能

### 2.1 设置

- 系统通知权限状态、授权与测试通知。
- 全部提醒暂停 30 分钟、1 小时或 2 小时，并可立即恢复。
- Windows 开机自动启动，可在界面中启停。
- 关闭主窗口后继续在后台和系统托盘运行。
- 托盘仅保留“打开摸个鱼”和“退出应用”。
- 本地 SQLite 数据健康状态与提醒数量展示。

### 2.2 提醒

- 创建固定时间提醒，支持工作日或每天重复。
- 创建按锚点对齐的间隔提醒，支持限定生效时段和跨午夜时段。
- 创建一次性提醒。
- 提醒列表展示规则摘要、下次触发时间和启用状态。
- 启用、停用和删除提醒。
- IANA 时区与夏令时边界校验。
- 主窗口关闭后，后台调度仍会按时运行。
- 启动、唤醒、解锁、系统时间变化和时区变化后自动重新核对到期提醒。
- 稳定 occurrence identity 和 SQLite 唯一约束防止重复投递。
- 循环提醒在暂停或勿扰期间直接跳过，不在恢复后批量弹出；一次性提醒可延期补发。

### 2.3 桌面悬浮框

- 桌面悬浮框是正常提醒的主投递通道。
- 悬浮框创建失败时，使用 Windows 系统通知兜底。
- 可见区域约为 310 × 118 像素，固定在主屏工作区右下角，不再居中遮挡工作内容。
- 使用透明无边框窗口，不抢占当前应用焦点。
- 咖啡角色和提示卡以弹跳动画进入，默认文案使用温和提醒语气。
- 提示展示 8 秒后自动淡出并关闭。
- 点击提示本身，或在提示出现后按下任意新按键，都会立即收起；按键只被观察，不会被拦截或吞掉。
- 自动、点击、按键和 `Alt+F4` 收起均将 occurrence 记录为“未处理”，避免误记为任务已经完成。
- 多条提醒同时到达时进入内存队列，按到达顺序逐条展示，不互相覆盖。
- 当前提示收起后自动展示队列中的下一条；队列为空时自动隐藏。
- 主窗口不存在时，悬浮框和后台调度仍可独立工作。

## 3. 主链路

```text
用户配置提醒
  -> SQLite 原子保存
  -> 后台调度计算到期时间
  -> 原子认领并执行暂停/会话策略
  -> 右下角桌面提示逐条展示
  -> 8 秒自动收起 / 点击收起 / 任意新按键收起
  -> 记录未处理并展示下一条
  -> SQLite 原子落库
```

投递失败链路：

```text
桌面悬浮框创建失败
  -> Windows 系统通知
  -> 两个通道均失败才记录 delivery_failed
```

## 5. 真机验收清单

以下项目需要在已解锁的 Windows 桌面手工确认，不属于新增功能开发：

1. 创建 1-2 分钟后触发的一次性提醒。
2. 关闭主窗口，确认托盘仍在且右下角提示按时出现，不抢输入焦点。
3. 等待 8 秒，确认提示自动淡出。
4. 再次触发提醒，点击提示，确认立即淡出。
5. 再次触发提醒，继续在原应用中键入任意新按键，确认提示消失且输入未丢失。
6. 同时创建两条同一分钟触发的提醒，确认逐条出现。
7. 对提示按 `Alt+F4`，确认下一条继续出现。
8. 重启应用，确认已结束的 occurrence 不重复弹出。
9. 开启开机启动后重新登录 Windows，确认应用仅在托盘后台启动。

MVP 代码功能至此冻结在“设置、提醒、桌面悬浮框”范围内。
