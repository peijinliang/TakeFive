# 摸个鱼 · TakeFive MVP 开发交付

**版本：** V1.1  
**日期：** 2026-07-15  
**状态：** MVP 主功能开发完成，待 Windows 桌面真机验收

---

## 1. MVP 范围

本版本只包含三个产品域：

1. 设置
2. 提醒
3. 桌面悬浮框

首次启动包含显式引导：用户可以按当前语言和系统时区启用推荐提醒，也可以跳过并从空列表开始。应用启动不会静默创建或启用默认提醒。

本版本不开发今日时间线、历史记录、统计报表、主题、账号、同步、导入导出或备份界面。底层已有但未进入这三条用户链路的能力不继续扩展。

## 2. 已完成功能

### 2.1 设置

- 系统通知权限状态、授权与测试通知。
- 全部提醒暂停 30 分钟、1 小时或 2 小时，并可立即恢复。
- Windows 开机自动启动，可在界面中启停。
- 关闭主窗口后继续在后台和系统托盘运行。
- 托盘仅保留“打开摸个鱼”和“退出应用”。
- 本地 SQLite 数据健康状态与提醒数量展示。

### 2.2 提醒

- 创建固定时间提醒，支持工作日或每天重复。
- 创建按锚点对齐的间隔提醒，支持限定生效时段和跨午夜时段。
- 创建一次性提醒。
- 提醒列表展示规则摘要、下次触发时间和启用状态。
- 启用、停用和删除提醒。
- IANA 时区与夏令时边界校验。
- 主窗口关闭后，后台调度仍会按时运行。
- 启动、唤醒、解锁、系统时间变化和时区变化后自动重新核对到期提醒。
- 稳定 occurrence identity 和 SQLite 唯一约束防止重复投递。
- 提醒列表返回结构化规则详情，由前端按当前语言生成摘要。
- 完成、延后 10 分钟、跳过和手动收起动作均通过 Occurrence 状态机落库。
- 循环提醒在暂停或勿扰期间直接跳过，不在恢复后批量弹出；一次性提醒可延期补发。

### 2.3 桌面悬浮框

- 桌面悬浮框是正常提醒的主投递通道。
- 悬浮框创建失败时，使用 Windows 系统通知兜底。
- 可见区域约为 326 × 194 像素，固定在主屏工作区右下角，不再居中遮挡工作内容。
- 使用透明无边框窗口，不抢占当前应用焦点。
- 用户主动点击后可以聚焦并使用键盘操作；显示时不会抢占正在输入的应用。
- 咖啡角色和提示卡以弹跳动画进入，默认文案使用温和提醒语气。
- 提示展示 60 秒后自动淡出并关闭。
- 提示提供完成、延后、跳过和收起按钮；自动超时和收起记录为“未处理”，不会把点击完成误记为未处理。
- 多条提醒同时到达时按到达顺序展示，队列 payload 先写入 SQLite，应用重启后可恢复未完成的悬浮投递。
- 当前提示收起后自动展示队列中的下一条；队列为空时自动隐藏。
- 主窗口不存在时，悬浮框和后台调度仍可独立工作。

## 3. 主链路

```text
用户配置提醒
  -> SQLite 原子保存
  -> 后台调度计算到期时间
  -> 原子认领并执行暂停/会话策略
  -> 右下角桌面提示逐条展示
  -> 完成 / 延后 / 跳过 / 60 秒超时收起
  -> SQLite 记录结果并展示下一条
  -> SQLite 原子落库
```

投递失败链路：

```text
桌面悬浮框创建失败
  -> Windows 系统通知
  -> 两个通道均失败才记录 delivery_failed
```

## 5. 真机验收清单

以下项目需要在已解锁的 Windows 桌面手工确认，不属于新增功能开发：

1. 首次启动确认不会自动创建提醒；分别测试启用推荐模板和跳过引导。
2. 创建 1-2 分钟后触发的一次性提醒。
3. 关闭主窗口，确认托盘仍在且右下角提示按时出现，不抢输入焦点。
4. 分别点击完成、延后、跳过和收起，确认结果和下一条状态正确。
5. 等待 60 秒，确认提示自动淡出并记录为未处理。
6. 同时创建两条同一分钟触发的提醒，确认逐条出现。
7. 在队列存在时重启应用，确认未完成的悬浮投递可恢复且不重复。
8. 对提示按 `Alt+F4`，确认下一条继续出现。
9. 开启开机启动后重新登录 Windows，确认应用仅在托盘后台启动。

MVP 代码功能至此冻结在“设置、提醒、桌面悬浮框”范围内。
